# 沟通管理计划 Communication Plan

## 沟通原则
- **主动汇报不过夜**：每个模块完成后立即通知干系人
- **CI绿灯是硬标准**：未通过CI的代码不交付
- **PM兜底**：所有风险和阻塞直接升级给PM，不积压

---

## 汇报机制

### 每日状态报告（当前 sprint）
- **形式**：飞书消息
- **频率**：每个子任务完成后自动触发
- **内容**：完成的功能、CI状态、遇到的问题
- **接收人**：项目发起人

### 里程碑报告
- **形式**：飞书消息 + 飞书文档
- **触发条件**：每个里程碑到达时
- **内容**：里程碑达成情况、剩余工作、风险状态

### 最终验收报告
- **形式**：飞书消息 + GitHub README
- **触发条件**：所有需求完成且CI绿灯
- **内容**：交付清单、运行说明、后续建议
